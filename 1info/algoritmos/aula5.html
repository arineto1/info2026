Dê "cérebro" ao seu programa. Aprenda a criar desvios no fluxo do código para que ele execute diferentes acções dependendo dos dados fornecidos.

1. O Fluxo de Decisão

SE (Condicional Simples)

Executa um bloco de comandos **apenas se** a condição for verdadeira. Se for falsa, o programa ignora o bloco e segue em frente.

SE-SENÃO (Condicional Composta)

Oferece dois caminhos: um para quando a condição é **Verdadeira** e outro obrigatório para quando é **Falsa**.

// Estrutura Básica no Visualg:

se (condição) entao

// comandos caso verdadeiro

senao

// comandos caso falso

fimse

2. Prática Comentada

A Condicional Simples: Verificação de Maioridade 

algoritmo "Verifica_Idade"
var
   idade : inteiro
inicio
   escreva("Indique a sua idade: ")
   leia(idade)
   
   // Se a condição for falsa, nada acontece no console
   se (idade >= 18) entao
      escreval("Acesso concedido. Bem-vindo ao sistema!")
   fimse
   
   escreval("Fim do programa.")
fimalgoritmo

B Condicional Composta: Par ou Ímpar? 

algoritmo "Par_Ou_Impar"
var
   num : inteiro
inicio
   escreva("Introduza um número inteiro: ")
   leia(num)
   
   // Usamos o resto da divisão (%) para testar a paridade
   se (num % 2 = 0) entao
      escreval("O número inserido é PAR!")
   senao
      escreval("O número inserido é ÍMPAR!")
   fimse
fimalgoritmo

C Sistema de Aprovação Escolar 

algoritmo "Situacao_Aluno"
var
   nota : real
inicio
   escreva("Digite a média final do aluno: ")
   leia(nota)
   
   se (nota >= 7.0) entao
      escreval("PARABÉNS! O aluno foi aprovado.")
   senao
      escreval("ESTUDE MAIS! O aluno foi reprovado.")
   fimse

3. Exercícios de Fixação

Ex 01 - Básico

Positivo ou Negativo: Peça um número ao utilizador e informe se ele é positivo (maior ou igual a zero) ou negativo.

Ex 02 - Comparação

Maior de Dois: Receba dois números inteiros. O programa deve exibir qual deles é o maior valor entre os dois informados.

Ex 03 - Segurança

Validação de Senha: Crie uma variável com uma senha definida (ex: "1234"). Peça ao utilizador para digitar a senha. Exiba "Acesso Permitido" ou "Acesso Negado".

Ex 04 - Aritmética

Divisão Segura: Receba dois números. Se o segundo for diferente de zero, exiba o resultado da divisão. Caso contrário, exiba "Erro: Divisão por zero!".

Ex 05 - Cotidiano

Multa de Trânsito: Peça a velocidade de um carro. Se for maior que 80km/h, exiba: "Você foi multado!". Se estiver no limite, exiba: "Boa viagem!".

Ex 06 - Lógica

Múltiplo de 3: Peça um número e informe se ele é múltiplo de 3. (Dica: Use o operador de resto % e verifique se o resultado é 0).

Ex 07 - Finanças

Limite de Crédito: Peça o salário de uma pessoa e o valor da prestação de um empréstimo. Se a prestação for maior que 30% do salário, diga "Empréstimo negado".

Ex 08 - Compras

Desconto Progressivo: Peça o valor total de uma compra. Se for maior que 100€, aplique 10% de desconto e mostre o valor final. Caso contrário, mostre o valor sem desconto.

Ex 09 - Desporto

Peso Pesado: Receba o peso de um lutador. Se for maior ou igual a 90kg, exiba "Categoria: Pesado". Caso contrário, exiba "Categoria: Leve".

Ex 10 - Desafio Final

Verificador Completo: Peça um número. O programa deve dizer se ele é PAR e ao mesmo tempo POSITIVO (use o operador 'e' dentro do SE).
