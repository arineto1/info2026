Aula 04: Operadores Relacionais e Lógicos

Ensine o seu algoritmo a comparar valores. É através da lógica que os sistemas decidem se um utilizador pode ou não aceder a uma funcionalidade.

1. A Arte de Comparar

Operadores Relacionais

  • Igual a =
  • Diferente de <>
  • Maior que >
  • Menor que <
  • Maior ou Igual >=
  • Menor ou Igual <=

Operadores Lógicos

E (AND):

Só é VERDADEIRO se ambas as condições forem verdadeiras.

OU (OR):

É VERDADEIRO se pelo menos uma das condições for verdadeira.

NAO (NOT):

Inverte o resultado. O que é Verdade vira Falso e vice-versa.

2. Prática Explicada

A Comparações Simples 

algoritmo "Testes_Relacionais"
var
   n1, n2 : inteiro
   teste : logico
inicio
   n1 <- 10
   n2 <- 20
   
   // Perguntamos: "O n1 é maior que o n2?"
   teste <- n1 > n2
   
   escreval("O valor 10 é maior que 20? ", teste)
   
   // Perguntamos: "O n1 é diferente do n2?"
   teste <- n1 <> n2
   escreval("O valor 10 é diferente de 20? ", teste)
fimalgoritmo

B Combinação com Operador Lógico "E" 

algoritmo "Acesso_Sistema"
var
   idade : inteiro
   tem_bilhete : logico
   pode_entrar : logico
inicio
   idade <- 20
   tem_bilhete <- verdadeiro
   
   // Para entrar, precisa ter 18 anos OU MAIS E ter o bilhete
   pode_entrar <- (idade >= 18) e (tem_bilhete = verdadeiro)
   
   escreval("O utilizador pode entrar no evento? ", pode_entrar)
fimalgoritmo

C A Liberdade do Operador "OU" 

algoritmo "Desconto_Cinema"
var
   estudante, idoso, tem_desconto : logico
inicio
   estudante <- falso
   idoso <- verdadeiro
   
   // Ganha desconto se for estudante OU se for idoso
   tem_desconto <- estudante ou idoso
   
   escreval("Este cliente tem direito a desconto? ", tem_desconto)
fimalgoritmo

3. Exercícios de Lógica Aplicada

01. Verificador de Meta:

Peça ao utilizador o valor das vendas do mês. Crie uma variável lógica que armazene se ele atingiu a meta (Vendas > 5000). Exiba o resultado lógico.

02. Identidade em Check:

Solicite dois nomes. Verifique se os nomes digitados são exatamente iguais e exiba o resultado (Verdadeiro/Falso).

03. Idade de Isenção:

No transporte público, crianças menores de 5 anos ou idosos com 65 anos ou mais não pagam. Receba a idade e diga se a pessoa tem isenção (Verdadeiro ou Falso).

04. Número no Intervalo:

Peça um número e verifique se ele está entre 10 e 20 (inclusive). Para isso, o número deve ser >= 10 E <= 20.

05. Sistema de Login:

Defina uma variável "senha_correta" com o valor "1234". Peça ao utilizador para digitar uma senha e mostre se o acesso foi concedido (Digitada = Correta).

06. Múltiplo de 5:

Receba um número e verifique se ele é múltiplo de 5. Dica: use o operador de resto % e verifique se o resultado é igual a 0.

07. Benefício Social:

Uma pessoa tem direito a auxílio se estiver desempregada OU se tiver renda menor que 1000 euros. Receba estas duas informações e exiba o direito ao benefício.

08. Validação de Triângulo:

Receba três lados (A, B, C). Verifique se eles podem formar um triângulo. A regra é: (A+B > C) E (A+C > B) E (B+C > A).

09. Aprovação Académica:

Para passar, um aluno precisa de Média >= 7.0 E Presença >= 75%. Receba os dois valores e mostre a situação lógica final.

10. Desafio Master: Votação Obrigatória

Crie um sistema que receba a idade de uma pessoa. Ela é obrigada a votar se tiver entre 18 e 70 anos (inclusive). Use os operadores relacionais e o operador "E" para validar.
